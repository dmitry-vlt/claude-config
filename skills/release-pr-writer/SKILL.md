# release-pr-writer

## Purpose
Use this skill to write crisp commit messages, PR descriptions, release notes, migration notes, and stakeholder summaries.

## Goals
Communicate change clearly for engineers, reviewers, operators, and customers.

## Output Types
- commit message
- PR description
- release notes
- migration note
- internal summary
- customer-facing summary

## Workflow
1. Identify audience.
2. Identify what changed.
3. Identify why it changed.
4. Identify risks / migration / rollback implications.
5. Write with clarity and brevity.
